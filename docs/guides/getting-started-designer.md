# Getting Started (Designer)

Audience: Designers working in Figma

This guide explains how the Figma Component DSL connects your Figma work with the codebase, and how design changes flow in both directions.

## What This Project Does for You

This project keeps your Figma components and the React codebase in sync:

- Components defined in code are **exported to Figma** as real, editable components with proper variants and properties.
- When you edit a component in Figma, those changes can be **exported as a changeset** and applied back to the code automatically.
- **Code Connect** links Figma components to their code implementations, so developers see the correct code snippets in Figma's Dev Mode.

## How Data Flows

```
  React Code                    DSL                       Figma
  ----------                  ------                    --------
      |                          |                          |
      |    [ compile + export ]  |                          |
      +------------------------->+--- [ plugin import ] --->|
      |                          |                          |
      |                          |    [ plugin export ]     |
      |<-- [ apply changeset ] --+<-------------------------+
      |                          |                          |
```

**Forward** (code to design): Developers write components, compile them to DSL, and export to Figma via the plugin.

**Reverse** (design to code): You edit components in Figma, export a changeset via the plugin, and the changeset is applied to the React/CSS source code.

## The Figma Plugin

The DSL plugin runs inside Figma and handles two main tasks:

### Importing components
The plugin reads a JSON file (generated by the CLI's export command) and creates real Figma components with:
- Correct auto-layout settings
- Proper fills, strokes, and corner radii
- Component properties (variants, text, booleans)
- Design token values

### Exporting changesets
After you edit a component in Figma, the plugin can export a **changeset** — a JSON file describing exactly what changed (e.g., "fill color changed from #000 to #333", "font size changed from 16 to 18"). This changeset is then applied to the code.

## Code Connect

Each component has a `.figma.tsx` file that maps Figma properties to React props. For example, a Button component maps:

| In Figma             | In Code                    |
|----------------------|----------------------------|
| Variant = "Primary"  | `variant="primary"`        |
| Size = "Large"       | `size="lg"`                |
| Label = "Click me"   | `children="Click me"`     |
| Full Width = true    | `fullWidth={true}`         |

This means Figma's Dev Mode shows developers the exact React code to use, with correct prop values based on the variant they're inspecting.

## The Changeset Workflow

When you make design changes and want them reflected in code:

1. **Edit in Figma** — change colors, typography, spacing, or other properties on a component
2. **Export changeset** — use the plugin's Export tab (Changeset mode) to generate a JSON file
3. **Share with developer** — the developer applies the changeset to update React/CSS code
4. **Verify** — the system renders both the Figma export and the React component, compares them pixel-by-pixel, and reports the match percentage

See [Syncing Design Changes](syncing-design-changes.md) for the full workflow.

## What You Should Know About Components

- Components with **variants** (e.g., Primary/Secondary, Small/Medium/Large) are stored as **component sets** in Figma. Each variant combination is a separate child within the set.
- **Design tokens** (colors, spacing, typography) are shared between Figma and code. Changes to tokens affect all components that use them.
- The **validator** checks components for compatibility before export. If a developer says a component "doesn't validate," it means the code structure needs adjustment before it can be sent to Figma.
