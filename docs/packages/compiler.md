## Overview
**Confidence**: 0.95 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

The `@figma-dsl/compiler` package translates DSL node trees (from `@figma-dsl/core`) into Figma-compatible `FigmaNodeDict` structures with full layout resolution. It provides two compilation paths: basic (metadata passthrough without geometry) and layout-aware (with a two-pass layout algorithm for precise sizing and positioning). The package integrates canvas-based text measurement via `@napi-rs/canvas` to compute text dimensions for auto-layout resolution.

## Public API
**Confidence**: 0.97 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

### Compilation Functions

#### `compile(node: DslNode): CompileResult`
Basic compilation without layout resolution. Converts DSL nodes to FigmaNodeDict tree with identity transforms. Useful for serialization without geometric layout.

#### `compileWithLayout(node: DslNode, measurer: TextMeasurer, options?: CompilerOptions): CompileResult`
Layout-aware compilation. Calls `resolveLayout()` to compute sizes and offsets, then applies resolved dimensions and 2D affine transforms to all nodes. Required for export-ready geometry. When `options.mode` is `'canvas'`, enables absolute positioning and passes through Canvas Mode properties (effects, blendMode, rotation, extended text styles).

#### `compileToJson(node: DslNode): string`
Convenience wrapper — calls `compile()` and returns `JSON.stringify(result, null, 2)`.

### Layout Resolution

#### `resolveLayout(root: DslNode, measurer: TextMeasurer, mode?: CompilerMode): LayoutResult`
Two-pass layout algorithm returning `{ sizes: Map<DslNode, ResolvedSize>, offsets: Map<DslNode, {x, y}> }`. When `mode` is `'canvas'`, frames without `autoLayout` use absolute positioning — children are placed at their declared `x`/`y` coordinates instead of (0, 0).

### Text Measurement

#### `textMeasurer.initialize(fontDir: string): void`
Registers TTF/OTF fonts from directory. Idempotent — skips if already initialized. Must be called before `measure()`.

#### `textMeasurer.measure(characters: string, style): TextMeasurement`
Computes text dimensions using canvas 2D context. Handles multi-line text. Returns `{ width, height }`.

#### `textMeasurer.measureWrapped(characters: string, maxWidth: number, style): TextMeasurement`
Computes text dimensions with word-wrapping at the given `maxWidth`. Splits on explicit newlines, then wraps each paragraph at word boundaries. Returns `{ width: maxWidth, height: totalLines × lineHeight }`. Used when `textAutoResize: 'HEIGHT'` is set with an explicit width.

#### `resetTextMeasurer(): void`
Clears initialization state. Used in tests.

**Evidence**: `src/compiler.ts:230-300`, `src/text-measurer.ts:17-58`, `src/layout-resolver.ts:30-45`

## Compilation Pipeline
**Confidence**: 0.94 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

### Basic Compilation Flow

```
DslNode (AST from @figma-dsl/core)
  ↓ compile()
  ├→ Reset GUID counter
  ├→ compileNode() recursive tree walk
  │   ├→ mapNodeType(): type mapping
  │   ├→ convertFill(): fill conversion
  │   ├→ convertStroke(): stroke conversion
  │   ├→ resolveAutoLayoutPadding(): padding expansion
  │   ├→ Text expansion: baselines, fontMeta
  │   ├→ Component properties → definitions
  │   └→ Instance overrides
  ↓
CompileResult { root: FigmaNodeDict, nodeCount, errors[] }
```

### Layout-Aware Compilation Flow

```
DslNode + TextMeasurer
  ↓ compileWithLayout()
  ├→ resolveLayout()
  │   ├→ measureNode() [bottom-up]: compute intrinsic sizes
  │   └→ positionChildren() [top-down]: calculate offsets
  ├→ compileNodeWithLayout()
  │   ├→ compileNode() for base structure
  │   ├→ Apply resolved sizes
  │   └→ Compose 2D affine transforms from offsets
  ↓
CompileResult { root: FigmaNodeDict (with geometry), nodeCount, errors[] }
```

### GUID Assignment
- Counter incremented per node: `[0, ++counter]`
- Deterministic within single `compile()` invocation (counter resets)
- Sequential: produces [0,1], [0,2], [0,3]...

## Auto-Layout Passthrough & Resolution
**Confidence**: 0.92 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

Auto-layout properties from `DslNode.autoLayout` are passed through to `FigmaNodeDict`:

| FigmaNodeDict Property | Source |
|----------------------|--------|
| `stackMode` | `al.direction` (HORIZONTAL/VERTICAL) |
| `itemSpacing` | `al.spacing ?? 0` |
| `primaryAxisAlignItems` | `al.align ?? 'MIN'` |
| `counterAxisAlignItems` | `al.counterAlign ?? 'MIN'` |
| `paddingTop/Right/Bottom/Left` | `resolveAutoLayoutPadding()` |
| `layoutSizingHorizontal/Vertical` | `al.widthSizing/heightSizing`, or inferred from `node.size` |

### layoutSizing Inference

When a node has auto-layout and an explicit `size` but no `widthSizing`/`heightSizing`, the compiler infers `FIXED`:
- If `node.size.x > 0` and `al.widthSizing` is not set → `layoutSizingHorizontal: 'FIXED'`
- If `node.size.y > 0` and `al.heightSizing` is not set → `layoutSizingVertical: 'FIXED'`

This ensures that frames like headers/footers with explicit dimensions (e.g., `size: { x: 1440, y: 64 }`) are imported into Figma at their specified size rather than collapsing to HUG mode.

### Padding Resolution (`resolveAutoLayoutPadding`)
Cascade order: per-side values (`padTop`, `padRight`, etc.) override shorthand values (`padX`, `padY`), which override default `0`.

## Layout Resolution Algorithm
**Confidence**: 0.92 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

### Pass 1: Bottom-Up Measurement (`measureNode`)

Computes intrinsic sizes for each node into `Map<DslNode, ResolvedSize>`:

1. **TEXT nodes**: Use `TextMeasurer.measure()` → `{width, height}`. When `textAutoResize: 'HEIGHT'` is set with an explicit `size.x`, uses `TextMeasurer.measureWrapped()` to compute height at the constrained width.
2. **Leaf shapes** (no children): Use explicit `node.size` or `{0, 0}`
3. **Containers with explicit FIXED size** (no autoLayout): Use `node.size`
4. **Auto-layout containers**:
   - **FIXED sizing**: Use `node.size`
   - **HUG sizing**:
     - Primary axis: sum of visible child sizes + spacing × (childCount - 1) + padding
     - Counter axis: max visible child size + padding
   - **FILL with no size**: Treated as HUG (conservative)

### Pass 2: Top-Down Positioning (`positionChildren`)

Computes child offsets into `Map<DslNode, {x, y}>`:

1. Calculate available space = container size - padding
2. Resolve FILL children: distribute remaining space equally among FILL/layoutGrow children
3. **Special case**: FILL inside HUG parent → treated as HUG (uses intrinsic size)
4. Apply primary axis alignment (MIN/CENTER/MAX/SPACE_BETWEEN)
5. Apply counter-axis alignment (MIN/CENTER/MAX)
6. Recurse into children

### Canvas Mode Absolute Positioning

When `mode` is `'canvas'` and a frame has no `autoLayout`, children are positioned using their declared `x`/`y` coordinates relative to the parent frame's top-left corner. Rotation is applied as a transform matrix rotation around each child's center point. Children without explicit `x`/`y` default to (0, 0). Standard-mode behavior remains unchanged.

### Key Behaviors
- **Visibility filtering**: `visible: false` children excluded from all layout calculations
- **SPACE_BETWEEN**: Recalculates spacing as `availableSpace / (childCount - 1)`
- **CENTER counter-axis**: `offset = (available - childSize) / 2`
- **Canvas Mode warnings**: Using `x`/`y` outside an absolute positioning context emits a warning (not error)

**Evidence**: `src/layout-resolver.ts:47-318`

## Text Measurement & Expansion
**Confidence**: 0.91 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

### Text Measurer (`text-measurer.ts`)

1. **Initialize** (once per process):
   - Scans `fontDir` for `.otf/.ttf` files
   - Registers all fonts under family name `'Inter'` (hardcoded)
   - Creates lazy 1×1 canvas singleton

2. **Measure**:
   - Font string: `"${fontWeight} ${fontSize}px ${fontFamily}"`
   - Splits by `\n`, measures each line with `context.measureText()`
   - Returns `{width: maxLineWidth, height: lineCount × lineHeight}`

### Text Expansion (`compileNode` for TEXT type)

For each TEXT node, generates:
- **textData**: `{characters, lines[]}` (split by `\n`)
- **textAutoResize**: Passthrough from `node.textAutoResize` or `style.textAutoResize` (if set)
- **derivedTextData**: `{baselines[], fontMetaData[]}`

**Baselines** (per line):
- `lineY`: `lineIndex × lineHeight`
- `lineHeight`: From textStyle or `fontSize × 1.2` default
- `firstCharIndex`, `endCharIndex`: Character range (accounting for `\n`)

**LineHeight resolution**:
- `unit === 'PERCENT'` → `(value / 100) × fontSize`
- `unit === 'PIXELS'` → `value`
- Default: `fontSize × 1.2`

**Evidence**: `src/text-measurer.ts:17-58`, `src/compiler.ts:156-198`

## Transform & Geometry
**Confidence**: 0.93 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

### Transform Format
3×3 affine matrix matching Figma's internal representation:
```
[[sx, shx, tx],
 [shy, sy,  ty],
 [0,   0,   1]]
```

### Basic Compilation
All nodes receive identity transform: `[[1, 0, 0], [0, 1, 0], [0, 0, 1]]`

### Layout-Aware Compilation
Parent transform `[a, b, tx, c, d, ty]` composed with child offset `{x, y}`:
- `tx' = a×x + b×y + tx`
- `ty' = c×x + d×y + ty`
- Result: `[[a, b, tx'], [c, d, ty'], [0, 0, 1]]`

Root node always has identity (no parent). Translation is additive through the tree.

## Error Handling
**Confidence**: 0.95 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

The compiler includes a `validateNode()` function that populates the `CompileError[]` array with validation warnings and errors during compilation:

### Validation Rules (`validateNode`)

| Check | Severity | Condition |
|-------|----------|-----------|
| Negative corner radius | Error | `cornerRadius < 0` |
| RGBA out of range | Warning | Any r/g/b/a value outside 0–1 |
| Non-positive stroke weight | Warning | `strokeWeight <= 0` |
| Non-positive font size | Error | `fontSize <= 0` |

Errors are returned in `CompileResult.errors[]` and logged to stderr by the CLI with `[nodeType] nodePath: message` format.

### Text Decoration Passthrough
The compiler preserves `textDecoration` from `TextStyle` into the compiled node, making it available for the renderer to draw underline/strikethrough lines.

## Performance Characteristics
**Confidence**: 0.88 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

| Operation | Complexity | Notes |
|-----------|-----------|-------|
| `compile()` | O(n) | Single recursive traversal |
| `compileWithLayout()` | O(n) | Layout O(n) + compilation O(n) |
| `measureNode()` | O(n) | Bottom-up, one pass |
| `positionChildren()` | O(n) | Top-down, one pass |
| Text measurement | O(lines) per node | Canvas `measureText()` per line |

- **Space**: O(n) for sizes and offsets Maps
- **Canvas**: Singleton 1×1 canvas, reused across calls (~1MB)
- **No caching**: Measurements and layouts recomputed on each `compile()` call

**Potential bottlenecks**: Text-heavy documents, deeply nested auto-layout hierarchies.

## Known Limitations
**Confidence**: 0.88 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

1. ~~**Error collection unused**~~: **Resolved** — `validateNode()` now populates errors for invalid cornerRadius, RGBA bounds, strokeWeight, and fontSize.
2. **Font family hardcoded**: `textMeasurer` registers all fonts as `'Inter'` regardless of file name. CJK text measurement uses the same font metrics.
3. **Letter spacing not measured**: `letterSpacing` parameter accepted but not applied to canvas `measureText()`.
4. **GUID scope**: Counter resets per `compile()` call — deterministic within a call, but concurrent compilations could conflict.
5. **No baseline font metrics**: Baselines computed from `fontSize × 1.2` default, not actual font metrics.
6. **Limited input validation**: `validateNode()` checks some properties (cornerRadius, RGBA, strokeWeight, fontSize) but does not detect missing properties, invalid fills, or circular references.
7. **Single-threaded**: All operations synchronous within the event loop.
