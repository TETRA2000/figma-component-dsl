## 1. Project Overview

**Confidence**: 0.97 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

Figma Design Playground is a SaaS landing page component library built with React 19, TypeScript 5.9, and Vite 8, featuring 16 production-ready UI components with CSS Modules for scoped styling. The project has a dual purpose:

1. **A runnable React application** — a complete landing page for a fictional SaaS product called "Prism"
2. **A Figma Code Connect integration layer** — each React component is paired with a `.figma.tsx` binding file that maps Figma design properties to React props, enabling Figma Dev Mode to display React code snippets

A companion Figma plugin programmatically generates the matching Figma component set, and a helper script synchronizes node IDs between the Figma file and the Code Connect bindings.

The application is entirely static — no backend, no API calls, no routing, no authentication, and no user-generated data.

## 8. Figma Code Connect Integration

**Confidence**: 0.94 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

### Configuration

`figma.config.json` declares the React parser with `include: ["src/components/**/*.tsx"]`.

### Binding Pattern

Each `.figma.tsx` file:
1. Imports `figma` from `@figma/code-connect` and the React component
2. Defines `FIGMA_URL` pointing to file key `18snbORL1E5bCSov2YontW` with a node ID
3. Calls `figma.connect(Component, FIGMA_URL, { props, example })`

### Property Mapping Helpers

| Helper | Used By | Purpose |
|--------|---------|---------|
| `figma.enum()` | Button, Badge, Container, Hero, FeatureGrid, Stats, LogoCloud, TestimonialCard | Figma variants → React string props |
| `figma.string()` | All 16 components | Figma text → React string props |
| `figma.boolean()` | Button, PricingCard | Figma toggles → React booleans |


### Known Limitation

Several Code Connect `example` functions pass empty arrays for child data (e.g., `features={[]}`, `testimonials={[]}`, `plans={[]}`, `logos={[]}`). Figma Dev Mode previews will show these components with no visible child content. (Validated: 3/3 confirmations)

## 9. Figma Plugin Architecture

**Confidence**: 0.91 | **Consensus**: Full | **Sources**: Architect, Developer, Analyst

### Purpose

Programmatically generates all 16 Figma components inside a Figma file, creating proper `ComponentNode` and `ComponentSetNode` objects with variants and properties matching the Code Connect bindings.

### Plugin Structure (`figma-plugin/code.ts`, ~1200-1647 lines)

1. **Color tokens** — Duplicated from `tokens.css` as hex constants
2. **Utility functions** — `hexToRGB()`, `solidPaint()`, `gradientPaint()`, `setAutoLayout()`, `createText()`, `createRect()`, `createIcon()`
3. **Component builders** — One async function per component
4. **Orchestrator** — Creates "Component Library" page, invokes all builders, arranges on canvas, outputs JSON node-ID mapping

### Variant Strategy

- **With variants**: `figma.combineAsVariants()` → `ComponentSetNode` (Button, Badge, Container, Hero, FeatureGrid, Stats, LogoCloud, TestimonialCard, PricingCard)
- **Without variants**: `figma.createComponent()` + `addComponentProperty()` (Navbar, FeatureCard, Testimonials, PricingTable, CTABanner, FAQ, Footer)

### Build & Usage

```bash
npm run plugin:build  # esbuild → figma-plugin/code.js (gitignored, ES2015)
# In Figma: Plugins > Development > Import manifest > Run
```

### Error Handling

The plugin has **no error handling** — no try/catch around font loading or component creation. If any operation fails, the entire plugin crashes with partial state potentially remaining on canvas. (Validated: 3/3 confirmations, one validator noted the situation is worse than initially described — `main()` has no `.catch()` handler at all)

> `figma-plugin/manifest.json:1-7`, `figma-plugin/code.ts`, `PLAN2.md:1-121`

### Input Format

### Behavior

- Hardcoded file key: `18snbORL1E5bCSov2YontW`
- No input validation on JSON (`JSON.parse` with no try/catch)

## 14. Design Patterns & Conventions

**Confidence**: 0.92 | **Consensus**: Majority | **Sources**: Architect, Developer

1. **Variant + Size union types** — CSS class selected by `styles[variant]` dynamic key access
2. **Manual class name composition** — `[...].filter(Boolean).join(' ')` (no `classnames`/`clsx`)
3. **Conditional rendering** — `{value && <Element>}` for optional content
4. **Spread props for cards** — `{items.map((item, i) => <Card key={i} {...item} />)}`
5. **Shared section header pattern** — Badge + h2 title + subtitle paragraph
6. **CSS custom properties for dynamic values** — FeatureGrid `--cols`
7. **Polymorphic Button** — `<a>` when `href` provided, `<button>` otherwise
8. **Index-based keys** — Used for static data lists (acceptable for non-reorderable data)

## 15. Security Considerations

**Confidence**: 0.87 | **Consensus**: Majority (validated) | **Sources**: Analyst + validators

- **XSS surface**: Effectively zero — all data is hardcoded, no user input. React's JSX escaping provides automatic protection.
- **Token exposure**: `$FIGMA_ACCESS_TOKEN` referenced from environment (correct approach, not hardcoded).
- **External link safety**: Footer correctly uses `target="_blank" rel="noopener noreferrer"` on social links.
- **Button anchor variant**: No `target`/`rel` attributes on the `<a>` variant. Currently safe (all hrefs are hash links) but would need attention for external URLs.
- **External resources**: 9 images loaded from `api.dicebear.com` with no CSP headers or SRI hashes.
- **Supply chain**: 3 runtime + 13 dev dependencies. `package-lock.json` pins versions. No audit configuration.

## 16. Performance Characteristics

**Confidence**: 0.85 | **Consensus**: Majority (validated) | **Sources**: Analyst + validators

- **Bundle**: Vite with tree-shaking. Lucide-react icons individually imported. CSS Modules generate unique class names at build time.
- **Scroll handler**: Fires on every scroll event (not throttled), but React's batching deduplicates identical state updates. Acceptable for the current use case.
- **LogoCloud marquee**: Duplicates logo array (`[...logos, ...logos]`) for infinite scroll. One-time allocation, GPU-composited `transform: translateX()`.
- **Navbar blur**: `backdrop-filter: blur(12px)` is the most expensive visual effect — forces compositing of everything behind the navbar.
- **Images**: 9 external images loaded eagerly (no `loading="lazy"`, no `srcset`, no explicit dimensions). Could affect LCP and CLS.
- **CSS transitions**: Card/button hover effects use `transform: translateY()` — GPU-composited, no layout/paint.
